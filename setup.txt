# Vaultora Project Setup and Installation Guide

## Backend Initialization and Dependencies

### 1. Initial Project Setup
The backend was initialized as a Node.js project (configured as an ES module with `"type": "module"`).

### 2. Core Dependencies
**Command used:** 
`npm install express mongoose dotenv cors @clerk/express json2csv`

**Why these dependencies are used:**
*   **`express`**: A minimalist web framework for Node.js, used to create the backend server and handle API routing.
*   **`mongoose`**: An Object Data Modeling (ODM) library for MongoDB, used to structure database schemas and interact with the database.
*   **`dotenv`**: Loads environment variables from a `.env` file into `process.env`, allowing secure management of configuration and secrets.
*   **`cors`**: Express middleware that enables Cross-Origin Resource Sharing, allowing the frontend application to communicate with the backend API securely.
*   **`@clerk/express`**: The official Clerk SDK for Express.js, used to implement secure user authentication, authorization, and session management.
*   **`json2csv`**: A library to convert JSON data (such as user transactions) into CSV format, facilitating data export functionalities.

### 3. Development Dependencies
**Command used:** 
`npm install -D nodemon`

**Why these dependencies are used:**
*   **`nodemon`**: A development utility that monitors the project directory for file changes and automatically restarts the Node server, greatly improving the developer experience.

## Frontend Initialization and Dependencies

### 1. Initial Project Setup
The frontend was initialized using Vite with the React template.

### 2. Core Dependencies
**Command used:** 
`npm install react-router-dom axios framer-motion lucide-react @clerk/clerk-react tailwindcss @tailwindcss/vite`

**Why these dependencies are used:**
*   **`react-router-dom`**: Enables declarative routing in React, used for navigating between different pages/components of the application.
*   **`axios`**: A promise-based HTTP client for the browser, used to make API requests to the backend server.
*   **`framer-motion`**: A production-ready motion library for React, used to create smooth animations and transitions for the UI.
*   **`lucide-react`**: A collection of clean, beautifully crafted SVG icons provided as React components, used for UI iconography.
*   **`@clerk/clerk-react`**: The official Clerk SDK for React, used to integrate secure user authentication, sign-in/sign-up components, and user management into the frontend.
*   **`tailwindcss`**: A utility-first CSS framework for rapidly building custom user interfaces directly in the markup.
*   **`@tailwindcss/vite`**: A Vite plugin for integrating Tailwind CSS V4 seamlessly into the Vite build process.

### 3. Data Fetching and Visualization Dependencies
**Command used:** 
`npm install @tanstack/react-query recharts`

**Why these dependencies are used:**
*   **`@tanstack/react-query`**: A powerful asynchronous state management library for React, used for declarative data fetching, caching, and updating of server state.
*   **`recharts`**: A composable charting library built on React components, used to create data visualizations like the expense trends chart.

## Mobile Initialization and Dependencies

### 1. Navigation, Networking, and UI Dependencies
**Command used:** 
`npm install @react-navigation/native @react-navigation/native-stack @tanstack/react-query axios lucide-react-native` (in the `mobile` directory)

**Why these dependencies are used:**
*   **`@react-navigation/native`**: The core library for routing and navigation in React Native applications, allowing navigation between different screens.
*   **`@react-navigation/native-stack`**: Provides a way for your app to transition between screens where each new screen is placed on top of a stack, using native navigation primitives for better performance.
*   **`@tanstack/react-query`**: A robust asynchronous state management library, used for declarative data fetching, caching, and updating of server state in the mobile application.
*   **`axios`**: A promise-based HTTP client, used to reliably make API requests to the backend server from the mobile application.
*   **`lucide-react-native`**: A comprehensive collection of beautifully crafted SVG icons optimized for React Native, used for consistent UI iconography.

### 2. Authentication and Secure Storage Dependencies
**Command used:** 
`npm install @clerk/clerk-expo expo-secure-store` (in the `mobile` directory)

**Why these dependencies are used:**
*   **`@clerk/clerk-expo`**: The official Clerk SDK for Expo and React Native. Used to provide secure user authentication, sign-in, and session management tailored for the mobile application.
*   **`expo-secure-store`**: An Expo library that provides a way to encrypt and securely store key-value pairs locally on the device. Required by Clerk to securely persist authentication tokens and session data.
